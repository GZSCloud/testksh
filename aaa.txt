[业务说明] 
1.调拨流程：
	1.1调拨单操作：
		1.1.1：创建调拨单
		1.1.2：删除单个调拨单
		1.1.3：批量删除调拨单
		1.1.4：修改调拨单信息
		1.1.5：获取登录用户所有还未发起调拨流程的调拨单信息
		1.1.6：获取所有登录用户的需要修改，即被驳回调拨单的信息
		1.1.7：获取登录用户正处于审批或者勾兑流程的调拨单信息
		1.1.8：获取登录用户创建的且已经结束调拨的调拨单信息
		1.1.9：获取当前登录用户需要审批处理的调拨单信息
		1.1.10：获取当前登录用户参与过审批的所有的调拨单信息
		1.1.11：获取当前登录用户需要勾兑处理的调拨单的信息
		1.1.12：获取当前登录用户参与过勾兑的所有的调拨单的信息
	1.2调拨单明细操作：
		1.2.1：重新插入调拨明细信息列表
		1.2.2：删除调拨单明细
		1.2.3：更新调拨明细数据
		1.2.4：获取某调拨单的所有明细信息
		1.2.5：根据调拨明细id获取调拨明细信息
		1.2.6：获取勾兑信息，包括可否完成的标志位以及调拨明细信息
	1.3流程操作：
		1.3.1：启动调拨流程
		1.3.2：完成任务节点
		1.3.3：批量完成任务
	1.4辅助
		1.4.1：获取登录用户可以调拨的仓库
		1.4.2：获取下一审批节点可选的用户集合
		1.4.3：获取驳回时可以跳转的节点
		1.4.4：获取调拨流程之前的操作信息，显示在审批页面
		1.4.5：获取调拨源仓库现有物料信息
[API说明]
1.1.1
简单描述：创建调拨单，同时准备进行调拨申请
请求地址：/allot/createAllotApply
请求方式：POST
请求参数：
{
    "allotDate": 调拨日期,
    "allotOper": 调拨人,
    "applyCause": 申请调拨原因,
    "applyDate": 申请时间,
    "applyOper": 申请用户,
    "sourceWarehouseId": 调拨源仓库,
    "targetWarehouseId": 调拨目的仓库
}
返回数据：
{
    "code": 状态,
    "msg": 返回信息,
    "data": null
}

1.4.1
简单描述：获取登录用户控制范围内的仓库，用于选择调拨仓库
请求地址：/allot/getStoreForAllot
请求方式：POST
请求参数：无
返回数据：
{
    "code": 状态,
    "msg": 返回信息,
    "data": null
}

1.1.2
简单描述：删除调拨单
请求地址：/allot/deleteAllotInfo
请求方式：POST
请求参数：
{
    "id": 调拨单id
}
返回数据：
{
    "code": 状态,
    "msg": 返回信息,
    "data": null
}

1.1.3
简单描述：批量删除调拨单
请求地址：/allot/deleteAllotInfoList
请求方式：POST
请求参数：
[
    {
        "id": 调拨单id
    }
]
返回数据：
{
    "code": 状态,
    "msg": 返回信息,
    "data": null
}

1.1.4

请求参数：
{

    "allotOper": 调拨人,
    "applyCause": 申请调拨原因,
    "applyDate": 申请时间,
    "applyOper": 申请用户,
    "sourceWarehouseId": 调拨源仓库,
    "targetWarehouseId": 调拨目的仓库
}
返回数据：
{
    "code": 状态,
    "msg": 返回信息,
    "data": null
}
